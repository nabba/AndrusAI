**Topic:** Distinguishing Authentication and Authorization

**When to use:**
Use this skill when designing access control, troubleshooting user access issues, or explaining system security requirements.

**Procedure:**
1.  **Authenticate (AuthN):** Verify identity using "something you know, have, or are."
2.  **Verify Factors:** Check credentials against stored data; for higher security, require Multi-Factor Authentication (MFA).
3.  **Authorize (AuthZ):** Once identity is confirmed, check the user's permission level against the resource requested.
4.  **Enforce Least Privilege:** Grant only the minimum access necessary for the verified user.
