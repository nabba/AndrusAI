# Email Delivery Capability

## Problem
The team can generate outputs but cannot deliver them via email. This blocks:
- Sending research summaries to stakeholders
- Delivering generated reports
- Notifying teams of completed tasks
- Sending alerts and monitoring notifications
- Weekly digest automation

## Solution: Add Gmail MCP Server

### Adding the Server
```
mcp_add_server:
  name: "gmail"
  query: "email gmail send email"
  env_vars: "GOOGLE_CLIENT_ID=xxx;GOOGLE_CLIENT_SECRET=xxx"
```

Note: Requires OAuth2 setup in Google Cloud Console with Gmail API enabled.

### Alternative: AgentMail (Simpler Setup)
```
mcp_add_server:
  name: "agentmail"
  query: "email gmail send email"
  env_vars: "AGENTMAIL_API_KEY=xxx"
```
- Purpose-built for AI agents
- Each agent gets its own inbox
- Simpler authentication

## Common Workflows

### 1. Send Research Report
```
1. Research crew gathers information
2. Writing crew formats as report
3. Send via email with:
   - Subject: "Research Report: [Topic] - [Date]"
   - Body: Formatted markdown converted to HTML
   - Attachments: PDF export if needed
```

### 2. Weekly Digest
```
1. Aggregate tasks completed this week
2. Summarize key findings and outputs
3. Send to distribution list
```

### 3. Alert Notifications
```
1. Monitor triggers (time-based, event-based)
2. Format alert message
3. Send to relevant stakeholders
```

## Email Formatting Best Practices

### Subject Lines
- Be specific and actionable
- Include date/time for recurring emails
- Use prefixes: [Report], [Alert], [Digest]

### Body Structure
```
## Summary
[Brief 2-3 sentence overview]

## Key Findings
- Finding 1
- Finding 2

## Details
[Full content]

## Next Steps
[Recommended actions]

---
Generated by AI Team | [Date]
```

### Attachments
- Use for long reports (>5000 words)
- Prefer PDF for formatting preservation
- Include summary in body when attaching

## Integration with Other Crews

### Research + Writing + Email
```
research_crew:
  - Gather data
  - Pass to writing_crew

writing_crew:
  - Format as report
  - Pass to email

email:
  - Send to recipients
```

### Monitoring Workflow
```
schedule: daily at 9am
workflow:
  - Check for new data/issues
  - Generate summary
  - Email to stakeholders
```

## Security Considerations

- Never send sensitive data without encryption
- Use BCC for distribution lists
- Review content before sending sensitive information
- Log all sent emails for audit trail
- Rate limit to avoid spam detection
