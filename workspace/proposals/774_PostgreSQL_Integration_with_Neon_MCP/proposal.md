# Proposal #774: PostgreSQL Integration with Neon MCP

**Type:** skill

## Why this is useful

Problem: The team lacks persistent structured storage for research outputs like contacts, policy data, and statistics. Without a database, data is lost between sessions and hard to query across multiple research iterations. Solution: This skill teaches how to set up Neon PostgreSQL via its MCP server, design schemas, perform CRUD operations, and integrate with research and writing crews. It includes example schemas for sales contacts, payment providers, and policy documents, as well as best practices for indexing, parameterized queries, and backups.
