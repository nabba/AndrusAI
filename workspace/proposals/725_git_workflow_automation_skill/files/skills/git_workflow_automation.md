# Git Workflow Automation for Agent Teams

## Purpose
Enable the coding and self_improvement crews to version control code, skills, and configurations. Track changes, enable rollback, and maintain audit trails for team improvements.

## When to Use
- After creating or modifying skills in skills/ directory
- Before and after code executor runs that modify files
- When experimenting with new capabilities (use branches)
- For audit logging of significant changes

## Core Workflows

### 1. Initialize Repository
```bash
git init
if [ ! -f .gitignore ]; then
echo -e "__pycache__/\n*.pyc\n.env\nnode_modules/\n" > .gitignore
fi
git add .gitignore
git commit -m "Initialize repository"
```

### 2. Automated Commit After Skill Creation
```bash
git add skills/
git diff --cached --quiet || git commit -m "Add skill: ${SKILL_NAME}

Description: ${SKILL_DESCRIPTION}
Generated by: self_improvement crew
Timestamp: $(date -Iseconds)"
```

### 3. Experimental Branch Workflow
```bash
# Create experiment branch
git checkout -b experiment/${EXPERIMENT_NAME}

# ... make changes ...

# If successful, merge back
git checkout main
git merge experiment/${EXPERIMENT_NAME} --no-ff -m "Merge experiment: ${EXPERIMENT_NAME}"

# If failed, discard
git checkout main
git branch -D experiment/${EXPERIMENT_NAME}
```

### 4. Change Audit Log
```bash
# Get all changes by self_improvement crew
git log --oneline --grep="self_improvement"

# Get recent skill additions
git log --oneline -- skills/

# Detailed diff for a commit
git show ${COMMIT_HASH}
```

### 5. Rollback Procedure
```bash
# View recent commits
git log --oneline -10

# Soft rollback (keep changes, undo commit)
git reset --soft HEAD~1

# Hard rollback (discard all changes)
git reset --hard HEAD~1

# Revert specific commit (safer)
git revert ${COMMIT_HASH}
```

## Commit Message Convention

Use structured commit messages for automated parsing:

```
<type>(<scope>): <subject>

[optional body]

[optional footer]
```

Types:
- `skill`: New or updated skill
- `code`: Code changes
- `config`: Configuration changes
- `fix`: Bug fixes
- `docs`: Documentation

Examples:
```
skill(pdf): Add PDF document processing pipeline

code(data-store): Add SQLite persistent storage tool

fix(circular-import): Resolve import cycle in utils
```

## Safety Rules

1. **Never force push** to shared branches
2. **Always branch** for experimental changes
3. **Commit atomically** - one logical change per commit
4. **Write descriptive messages** - future agents need context
5. **Test before merging** - use code_executor to validate

## Python Helper Functions

```python
import subprocess
from datetime import datetime

def git_commit(path: str, message: str, agent: str = "agent") -> bool:
    """Stage and commit changes."""
    subprocess.run(["git", "add", path], check=True)
    result = subprocess.run(
        ["git", "commit", "-m", f"{message}\n\nAgent: {agent}\nTime: {datetime.now().isoformat()}"],
        capture_output=True
    )
    return result.returncode == 0

def git_create_branch(name: str) -> bool:
    """Create and checkout new branch."""
    result = subprocess.run(["git", "checkout", "-b", name])
    return result.returncode == 0

def git_status() -> str:
    """Get current git status."""
    result = subprocess.run(["git", "status", "--short"], capture_output=True, text=True)
    return result.stdout
```

## Pre-Commit Checks

Before committing new skills:
1. Validate YAML/Markdown syntax
2. Check for required skill sections
3. Verify no secrets or credentials
4. Run any defined tests

```bash
# Pre-commit validation
python -c "
import yaml, sys
for f in skills/*.md:
    try:
        content = open(f).read()
        assert '## Purpose' in content, f'Missing Purpose in {f}'
        print(f'{f}: OK')
    except Exception as e:
        print(f'{f}: FAILED - {e}')
        sys.exit(1)
"
```
