# Email Communication Capability Skill

## Purpose
Enable the team to send emails, manage inbox, and deliver automated reports to stakeholders.

## Gap Addressed
The team has `time-based_email_counting_precision` skill but no email sending capability. Cannot deliver results, send notifications, or communicate externally.

## Required MCP Server
Use `mcp_add_server` with one of:
- **Gmail** (name: `gmail`, query: `email send`) - Full Gmail integration
- **AgentMail** (name: `agentmail`, query: `email send`) - AI-native email API

## Core Capabilities Gained

### Sending Emails
- Send plain text and HTML emails
- Add attachments (reports, images, data)
- Set recipients (To, CC, BCC)
- Custom subject lines and headers

### Draft Management
- Create drafts for review
- Update existing drafts
- Schedule sends (Gmail)

### Inbox Management
- Read and search messages
- Filter by label, sender, date
- Mark as read/unread, archive, trash

### Automation Patterns
- Send daily/weekly digest emails
- Alert on task completion
- Forward important findings
- Reply to specific threads

## Common Workflow Patterns

### Pattern 1: Automated Report Delivery
```
1. Research crew gathers data
2. Writing crew creates report
3. Code executor generates PDF/HTML
4. Email crew sends to stakeholders
5. Log delivery in memory
```

### Pattern 2: Task Notification
```
1. Self-improvement crew identifies issue
2. Create detailed analysis
3. Email relevant team members
4. Track response in memory
```

### Pattern 3: Scheduled Digest
```
1. Aggregate findings from memory
2. Format as HTML newsletter
3. Send to distribution list
4. Archive sent message
```

## Authentication Requirements
- **Gmail**: OAuth2 credentials or App Password
- **AgentMail**: API key (USDC-based)
- Set via env_vars when adding server

## Integration with Existing Tools
- `web_search`: Research content for emails
- `file_manager`: Attach generated reports
- `memory_store`: Track sent emails and responses
- `writing` crew: Compose email content

## Rate Limiting & Best Practices
1. Respect daily send limits (Gmail: ~500/day)
2. Use batches for bulk notifications
3. Include unsubscribe option for recurring emails
4. Test with drafts before sending
5. Archive processed emails to avoid re-sending

## Security Considerations
- Never include sensitive credentials in email body
- Use BCC for distribution lists
- Verify recipient addresses before sending
- Log all outbound communications
